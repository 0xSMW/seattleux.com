# Heuristic evaluation checklist

## Visibility of system status

- [ ] Progress indicators exist for long operations
- [ ] Success and failure states are clear

## Match between system and real world

- [ ] Language matches user vocabulary
- [ ] Concepts are grouped logically

## User control and freedom

- [ ] Undo/cancel where appropriate
- [ ] Safe exits from flows

## Consistency and standards

- [ ] Terminology and patterns are consistent
- [ ] Common actions behave the same everywhere

## Error prevention and recovery

- [ ] Constraints are communicated early
- [ ] Errors explain how to recover

- [ ] Keyboard navigation works
- [ ] Focus is visible
