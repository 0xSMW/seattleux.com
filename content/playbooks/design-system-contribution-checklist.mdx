# Design system contribution checklist

## Before proposing

- [ ] Confirm there’s a real reuse need across teams
- [ ] Check existing components/patterns
- [ ] Identify accessibility requirements

## Spec and design

- [ ] Variants and states defined (hover, focus, disabled, error)
- [ ] Content guidelines included
- [ ] Responsive behavior defined

## Implementation

- [ ] Keyboard behavior defined and implemented
- [ ] Accessible names/labels included
- [ ] Tests or examples included

## Documentation

- [ ] “When to use” and “When not to use”
- [ ] Usage examples
- [ ] Migration notes if replacing something
